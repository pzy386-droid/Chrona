# Chrona 更新日志

## v0.3.0 - 2026-06-11

### 版本主题
- 本次是继 `v0.2.0` 之后的一次主线大更新：Chrona 从“规则解析 + 时间块调度原型”推进到“DeepSeek AI 草稿解析 + Motion 风格时间块交互”的阶段。
- 产品方向继续保持 Calendar First：AI 只负责理解意图和生成草稿，用户确认后才写入 SQLite，Scheduler 仍然负责最终时间块落位和冲突校验。

### 新增
- 接入 `AIService` / `DeepSeekProvider`：Quick Add 会优先调用 DeepSeek 解析自然语言任务。
- 新增 DeepSeek 规划草稿弹窗：AI 返回标题、截止时间、预计时长、优先级、分类、备注、时间锚点和解释，确认后才加入日程。
- 支持 AI 直接生成 Time Block：例如“后天早读”“明天中午吃饭”会生成带起止时间的时间块，而不是只生成任务 DDL。
- 新增 AI 接入设置能力：API Key 保存到本地 `app_settings`，不写入仓库。
- Time Block 选中后支持轻量 handle：
  - 上边缘拖拽：调整开始时间；
  - 下边缘拖拽：调整结束时间；
  - 右边缘拖拽：调整连续天数。
- 支持连续时间块整组上下 resize：跨天合并块调整起止时间时，会同步更新整组每天同一时段，避免被拆成两块。

### 改进
- Quick Add 改为异步请求 DeepSeek，等待 AI 返回时显示居中的旋转加载动画，避免主界面卡住。
- DeepSeek 草稿弹窗升级为深色玻璃化风格，并加入轻微呼吸感，匹配 Chrona 的高级深色 UI。
- AI 草稿确认时保留 `scheduledStart / scheduledEnd / hasTimeAnchor`，修复“左侧日期正确但时间轴落到今天”的前后端不一致。
- 创建 AI 时间块后会自动选中新建的具体 Time Block，右侧详情和时间轴指向同一个对象。
- 时间轴最小行距支持完整 24 小时视图，低密度下小时刻度调整为 2 小时间隔。
- 小高度时间块启用紧凑排版，标题、时间和 handle 会自适应尺寸，避免文字跑出卡片。
- 右拉跨天改为连续视觉预览，跨过临界点时播放右缘“钻入下一天”的吸附光效，松手后再一次性提交到后端。
- Time Block 宽高变化增加缓动，减少跨天延展和上下 resize 时的生硬跳变。

### 修复
- 修复 DeepSeek 请求期间 UI 卡顿的问题。
- 修复 AI 草稿中“备注”占位文字显示为黑色、深色主题下不可读的问题。
- 修复“后天/明天”类自然语言创建后左侧任务日期和实际时间轴位置不一致的问题。
- 修复右拉跨天松手时因 QML delegate 与 C++ model reset 同时发生导致卡退的问题。
- 修复连续时间块上下 resize 后只修改首日块、导致合并块裂开的问题。
- 修复最小行距下小时间块 handle 过粗、视觉比例不协调的问题。
- 修复短时间块标题溢出到时间块外的问题。

### 架构说明
- AI 仍然保持解耦：`AIService` 只生成可确认草稿，不直接写数据库。
- Scheduler 和 SQLite 持久化仍是真实数据源；QML 只负责展示和交互。
- 连续跨天时间块仍按“每天一个真实 TimeBlock”存储，Timeline 渲染层自动合并同任务、同时间段、连续日期的块，兼顾数据清晰和 Motion 风格视觉表现。

## v0.2.0 - 2026-06-01

### 产品定位
- 继续向 Motion 风格的 AI-native 学习时间调度系统推进：Calendar First、Focus First，弱化传统任务列表。
- 时间块成为核心交互对象，固定/非固定只是同一个时间块的状态，不再拆成两套用户心智。

### 新增
- 支持连续日期时间块：在任务详情中选择连续天数后，时间轴会以横向延展的方式显示为一个跨天时间块。
- 当前专注区域内联显示调度冲突，并提供重规划入口，减少顶部信息层占用。
- 新增晚间复盘入口，用于查看当天专注与完成概览。

### 改进
- 统一普通学习块与固定时间块的选中、hover、锁定图标和详情面板体验。
- 固定开关改为编辑态：点击开关不会立刻只修改某一天，保存时才会应用到整个连续时间块范围，避免跨天块分裂。
- 删除任务详情中的“截止时间”输入，时间调整统一通过当前时间块的起止时间完成。
- 顶部欢迎区域压缩为更紧凑的工具条，为日历时间轴释放更多垂直空间。
- AI 推荐、Quick Add、OCR 入口与容量健康合并到同一行，减少卡片堆叠。
- 移除 `Qt5Compat.GraphicalEffects` 依赖，避免缺少 QML 模块时应用无法启动。

### 修复
- 修复取消固定后连续时间块被拆成多个独立块的问题。
- 修复固定时间块点击后视觉反馈不一致的问题。
- 修复手动安排任务块晚于旧截止时间时，任务仍被 Scheduler 判定无法排入的问题。
- 修复“结束今日”按钮在侧栏中尺寸过小、文字被裁切的问题。
- 修复队友新增弹层中的 QML 字符串/调试可见状态导致应用启动失败的问题。

### 兼容性说明
- 底层仍保留按天存储时间块/固定事件，时间轴渲染层会自动合并连续同时间段的块，保证视觉上呈现为一个跨天时间块。
- AI 接入接口继续保留，当前版本重点完成调度 UI 与时间块交互主线。
